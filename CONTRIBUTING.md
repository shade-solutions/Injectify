## Getting Started

### Prerequisites

- [Git](https://git-scm.com/)
- [Node.js](https://nodejs.org/) ≥ 18 (for icon generation and future tooling)
- Google Chrome (or Chromium)

### Setup

```bash
# 1. Fork the repo on GitHub, then clone your fork
git clone https://github.com/<your-username>/Injectify.git
cd Injectify

# 2. Install development dependencies
npm install

# 3. Generate extension icons (required after a fresh clone)
node scripts/generate-icons.js

# 4. Load the extension in Chrome
#    chrome://extensions → Developer mode ON → Load unpacked → select /extension
```

## Project Structure

```
extension/          Chrome extension source — load this as "unpacked"
  manifest.json     Extension manifest (Manifest V3)
  background.js     Service worker: storage, message routing, URL matching
  content.js        Content script: injects JS / CSS / CDN per matching rule
  popup.*           Toolbar popup (quick overview of active rules)
  options.*         Full dashboard (add / edit / delete / import / export rules)
  icons/            PNG icons generated by scripts/generate-icons.js

docs/               Static landing page deployed to GitHub Pages
  index.html        Single-file landing page (no build step required)

assets/             Source assets
  icon.svg          Master SVG icon (source for PNG icons)

scripts/            Developer utilities
  generate-icons.js Generates extension/icons/ from source SVG

.github/workflows/  CI/CD
  deploy-docs.yml   Deploys /docs to GitHub Pages on push to main
```

## Development Workflow

1. **Create a branch** from `main`:
   ```bash
   git checkout -b feat/my-feature
   # or
   git checkout -b fix/my-bug-fix
   ```

2. **Make your changes.**
   - Keep changes focused and minimal.
   - Test them by reloading the extension in Chrome (`chrome://extensions` → ↺ reload button).

3. **Regenerate icons** if you changed `assets/icon.svg`:
   ```bash
   node scripts/generate-icons.js
   ```

4. **Commit** with a clear, imperative message:
   ```
   feat: add per-rule keyboard shortcut
   fix: correct subdomain wildcard matching
   docs: update URL pattern reference table
   ```

5. **Push** your branch and open a Pull Request on GitHub.

## Pull Request Guidelines

- **One concern per PR.** Small, focused PRs are reviewed faster.
- **Describe what and why** in the PR description.
- **Link related issues** using GitHub keywords: `Fixes #123`, `Closes #456`.
- Ensure the extension loads without errors in Chrome before submitting.
- If you add a new feature, update `README.md` and/or `docs/index.html` accordingly.
- Be responsive to review feedback — maintainers may request changes.

---

## Code Style

- **Language:** vanilla JavaScript (ES2020+), no TypeScript, no bundler.
- **Format:** 2-space indentation, single quotes for strings.
- **`'use strict'`** at the top of every JS file.
- **No external runtime dependencies** in the extension itself (`node_modules` is dev-only).
- Comment non-obvious logic; avoid comments that merely restate the code.
- HTML and CSS follow existing conventions in the codebase.

## Reporting Bugs

Please [open a GitHub issue](https://github.com/shade-solutions/Injectify/issues/new) and include:

- Chrome version
- Extension version (visible in `chrome://extensions`)
- Steps to reproduce
- Expected vs actual behaviour
- Console errors (if any) — open DevTools on the offending page or inspect the extension's
  background service worker
